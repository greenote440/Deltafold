## Executive Summary

After approximately 30 epochs of training across multiple configurations, the model achieves a genuine and reproducible Cluster ARI of ~0.39–0.41 (it demonstrably organises proteins by fold family), yet the TM-score rank correlation ρ remains statistically indistinguishable from zero (ρ ≈ 0.003, p = 0.94). These two facts are not contradictory. They point to a fundamental mismatch between the metric we are training on (binary cluster membership) and the metric we are evaluating on (continuous structural similarity). The shortcut-removal work was necessary and largely successful; the ceiling we have hit is architectural and objective-level, not a data or engineering problem.

---

## 1. What ARI Measures vs What ρ Measures

Understanding why these two numbers diverge is the most important result of this session.

### 1.1 Cluster ARI
The ARI metric used here runs spherical k-means (K = 49) on the 101 multi-member-cluster proteins in the validation split, then computes the Adjusted Rand Index between the k-means assignments and the ground-truth cluster labels from `cluster.tsv`. A score of 0 means random (no better than chance); 1 means perfect recovery.

`cluster.tsv` was generated by Foldseek, which combines the 3Di structural alphabet with a TM-score cutoff (~0.4). Proteins within the same cluster are structurally similar by Foldseek's definition. An ARI of 0.39–0.41 means the model has learned to place Foldseek-similar proteins near each other in embedding space. **This is real learning.**

### 1.2 TM-Score Rank Correlation ρ
`evaluate_correlation.py` randomly samples 30 proteins, runs pairwise TM-align (tmtools) on all 435 pairs to get actual TM-scores, and computes the Spearman rank correlation between those TM-scores and the model's pairwise cosine distances. The question is not "do similar proteins cluster together?" but "is the embedding distance metric monotone with structural similarity across the full continuous range of TM-scores?"

### 1.3 Why They Diverge: The Binary vs Continuous Problem

Supervised Contrastive Learning (SupCon) optimises a single binary signal: *push same-cluster proteins together; push different-cluster proteins apart.* It does not care about:

- The TM-score between two proteins **within** the same cluster (0.41 vs 0.95)
- The TM-score between two proteins **across** clusters (0.05 vs 0.38)
- Whether the continuous ordering of distances inside and outside clusters matches TM-score

A model that perfectly solves SupCon would produce embeddings where same-cluster proteins have cosine distance ≈ 0 and different-cluster proteins have distance ≈ 1. The Spearman correlation between that step-function distance and TM-score is 0 by construction, because TM-score varies continuously within both groups.

**ARI rising while ρ stays near 0 is the expected outcome of SupCon, not evidence of a bug.**

## 3. Interventions Tried, In Order

### 3.1 Baseline (pre-session)
**Config**: asymmetric model, standard NT-Xent (InfoNCE), cluster-aware split, all node features (AA + 3Di + PE + φ/ψ), symmetric contrastive, random batches.

**Result**: ρ ≈ 0.01 (not significant). Composition ablation shift = 0.015 (composition barely used). Sequential attention dominant (seq ρ = 0.36 > 3D ρ = 0.19).

---

### 3.2 Shortcut Diagnostic Battery
Built `diagnostics.py` with ten tests from the shortcut analysis report. This revealed the active shortcuts at baseline. The key finding: the model was NOT using composition (shift = 0.015), so removing residue features was not yet necessary. The main culprits were sequential attention and easy negatives.

---

### 3.3 First Mitigation Attempt: Unsupervised + PE Off + Hard-Neg-Beta + Phylo Split
**Command**:
```
--no-positional --unsupervised --hard-neg-beta 1.0 --split phylo
```

**What changed from baseline**: Removed sequence-index positional encoding (PE), switched from cluster-label SupCon to pure instance discrimination (InfoNCE), added loss reweighting toward hard negatives, switched train/val split to be taxonomically disjoint.

**Result**: ρ ≈ 0.01 (unchanged). Composition ablation WORSENED dramatically: AA shift = 0.358 (was 0.015), AA+3Di shift = 0.723.

**Diagnosis**: Removing supervised labels eliminated the only signal tying embeddings to fold. The model fell back to the cheapest view-invariant fingerprint available: amino-acid and 3Di composition. Both augmented views of the same protein share composition (jitter doesn't touch residue identity; 15% masking leaves 85% intact), so InfoNCE is solved trivially by composition without any structural information. This is the §2.2 composition shortcut, latent in the original model but now dominant.

**Lesson**: Switching to unsupervised without simultaneously removing composition channels invites the composition shortcut. The PE removal was correct but insufficient alone.

---

### 3.4 Geometry-Only Attempt: Unsupervised + No Composition + Hard-Neg-Mining
**Command**:
```
--no-positional --no-residue --no-3di --unsupervised --hard-neg-beta 1.0
--hard-neg-mining --split phylo
```

**What changed**: Removed all composition channels (AA, 3Di) forcing the model to use backbone geometry only (φ/ψ angles, Cα contacts, SSE shape, global PCA descriptors).

**Engineering problems encountered before training could be evaluated**:
- MPS crash (exit 133) on step 2 due to memory cliff: a large batch reached ~17k residues (original) → ~269k edges → several GB of retained attention activations → disk swap on 16 GB unified memory.
- Fixed by: `--max-residues 4000` (residue budget cap in sampler).
- Further creep from variable batch shapes causing MPS kernel recompilation each step (~34s/step).
- Fixed by: bucket-padding (pad nodes/edges/SSEs/proteins to multiples of 1024/256/8), bringing sustained step time to ~0.9s.

**Result after engineering fixes**: ρ ≈ 0.003. ARI ≈ 0 (model did not learn fold clusters without composition or supervised labels after a few epochs).

**Diagnosis**: With no composition, no supervised labels, and easy random negatives even within length-bucketed batches, the model had no tractable signal to latch onto. Hard-negative batch construction helps but cannot force the model to discover fold topology from geometry alone in a handful of epochs.

### 3.5 Supervised SupCon Restored: Current Best Configuration
**Command** (evolved over multiple restarts):
```
--no-positional --no-residue --hard-neg-mining --split phylo --hard-neg-beta 1.0
```
(3Di kept; supervised cluster-label SupCon; bucket-padded; phylo split)

**Bugs fixed during this run**:
1. `--hard-neg-beta` was silently ignored on the supervised path (the loss function dispatcher called `supervised_ntxent_loss` which had no `hard_neg_beta` parameter, bypassing `NTXentLoss` entirely). Fixed by adding `hard_neg_beta` to `supervised_ntxent_loss`.
2. Checkpoint saving used only val loss, which became unreliable after `hard_neg_beta` changed the loss scale (val loss jumped from 0.99 to ~2.0 between restarts). Fixed by checkpointing on max(ARI, val-loss improvement).

**ARI trajectory** (5-seed average, 49 clusters, 101 val proteins):
```
epoch  1: 0.2787
epoch  2: 0.3242
epoch  7: 0.3574  ← first new best after hard_neg_beta fix
epoch 10: 0.3857
epoch 20: 0.3922
epoch 26: 0.4057  ← best so far
epoch 31: 0.3827  (run ongoing)
```

**Val loss trajectory**: 1.97 → 1.16 (still declining at epoch 31). Model is still learning.

**ρ on best checkpoint (epoch 26/31)**: 0.003, p = 0.94. Not significant.

## 5. Persistent Problems Not Solved

### 5.1 ρ ≈ 0 Despite ARI ≈ 0.40

The core reason is stated in §1.3: SupCon is a binary objective. It can teach the model to group fold-families together but not to rank by TM-score. ARI and ρ measure different things, and achieving one says little about the other.

Additionally: Foldseek clusters (used as SupCon positive labels) are themselves built from 3Di + TM-score at a threshold. The model is learning to replicate Foldseek's binary clustering, not the underlying raw TM-score distribution. This is circular — we are teaching it to reproduce an approximation of what we want to measure.

### 5.2 Sequential Attention Not Fully Fixed

After removing the positional encoding, the attention diagnostic shows:
- `|corr(attn, |i−j|)|` = 0.39 (sequence separation)
- `|corr(attn, 3D dist)|` = 0.33 (structural distance)

Sequence still marginally dominates. Removing PE helped (was 0.35 vs 0.19) but did not invert the relationship. The residual sequential bias likely comes from the fact that Cα coordinates themselves encode sequence position for locally structured regions (helices, strands have monotone coordinate sequences). This cannot be fixed by removing PE alone.

### 5.3 Step Time Creep in Long Runs

Despite bucket-padding (which bounds unique MPS kernel shapes), step times still drift from ~1.7s/it (epoch 7) to ~4.9s/it (epoch 29). This is the MPS Metal shader cache accumulating internal compiled-graph state that is not exposed to user-level APIs (`empty_cache()` only flushes the allocator, not the compiled kernel registry). The only available mitigation is periodic cold restarts from checkpoint (every 20–25 epochs), which reset the Metal state.

### 5.4 ARI Metric Variance

Even with 15-seed averaging, ARI swings ±0.05–0.10 epoch to epoch on 49 clusters / 101 proteins. K=49 on 101 points is a fundamentally hard k-means problem. The best-ARI checkpoint selection partially compensates, but it makes it difficult to detect genuine plateaus vs. noise.

## 6. What ARI 0.40 Actually Means

ARI of 0.40 on 49 Foldseek-derived clusters means the model has learned to partially recover the same fold-family organisation that Foldseek finds. This is non-trivial: the evaluation uses a phylogenetically held-out val set (taxa not seen during training), so the model is generalising to novel viral families.

What it does NOT mean:
- That embedding distances are monotone with TM-score (ρ = 0)
- That the model can rank proteins by structural similarity
- That it has learned the geometry of fold space; it may have learned a coarser "fold family identity" signal

## 7. What Would Likely Move ρ

In order of expected impact:

1. **TM-score regression auxiliary loss** (`--tm-aux-weight`): Already implemented. Regresses `cos(z_i, z_j)` against `2·TM − 1` on sampled in-batch pairs. Not used in this session due to per-step tmtools alignment cost (~0.5s/pair). Could be made tractable by computing only on pairs where TM-score is already available from cluster construction.

2. **Pre-computed TM-score labels for all cluster pairs**: Store actual TM-scores for within-cluster pairs, use them as continuous targets in a regression or soft-label InfoNCE. The 18,192 clusters in the paper have pairwise TM-scores already computed; these could replace binary cluster membership as the positive weighting.

3. **A continuous contrastive loss**: Replace SupCon (binary: same cluster vs not) with a loss that weights positives and negatives by their actual TM-score. E.g. soft InfoNCE where the target distribution is proportional to TM-score rather than a hard cluster assignment.

4. **Larger training set**: 3,076 training proteins is small. The full 67,715-structure dataset would provide more fold diversity and more cluster co-occurrences per batch.

5. **Structural data quality**: All structures are ColabFold predictions. Low-pLDDT regions introduce systematic fallback conformations. Masking residues with pLDDT < 70 before computing the PCC (topotein_lifter hook points are implemented) would reduce noise in the geometry signal.

---

## 8. Recommendation

The model in its current form is a **fold classifier**, not a **structural distance metric**. ARI ~0.40 on held-out taxa is a useful result for taxonomic/functional annotation tasks. If the goal is a continuous TM-score-correlated embedding, the path requires either:

(a) Adding a continuous supervision signal (TM-score regression auxiliary or soft-label contrastive), or  
(b) Accepting that contrastive learning with binary cluster labels does not directly optimise TM-score, and switching to a different objective (e.g. direct metric learning with pre-computed TM-scores as distances).

Continuing to train the current configuration to 100 epochs will likely raise ARI further (val loss is still declining) but will not move ρ, because ρ is not in the loss.
